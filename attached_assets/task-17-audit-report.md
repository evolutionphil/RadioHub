# Task #17 — Bing ContentQuality Cleanup (51 URLs)

Source CSV: `themegaradio.com_SitemapIndexCoverageIssueDetail_4_22_2026_1776870207368.csv`
(51 distinct URLs, all flagged `NotIndexedAndMayNeedAttention/ContentQuality`).

This report describes how each root cause is addressed by the new code in
this task. The per-station audit row CSV is generated by running
`tsx scripts/clean-content-quality-urls.ts` and written to
`attached_assets/task-17-audit-report.csv`.

## Root causes & fixes

| # | Root cause | URLs in CSV | Fix shipped in this task |
|---|---|---|---|
| 1 | Slug stripped to bare punctuation because `[^\w\s-]` regex dropped Arabic / Cyrillic / Thai / accented Latin chars | ~12 (e.g. `/th/สถานี/...`, `/gu/સ્ટેશન/...`, `/ko/스테이션/...`) | New `slugifyStationName()` in `server/seo/junk-station-rules.ts` uses `transliteration` to produce ASCII slugs; both copies of the slug generator in `server/routes/slug-routes.ts` and the inline one in `server/routes/station-public-routes.ts` now call it. The migration script preserves the previous slug in `slugAliases[]` for 301 fallback in `server/seo-renderer.ts`. |
| 2 | Test-feed records (`-dolby-atmos-test-1`, etc.) | 1 (`/fi/asema/ard-dolby-atmos-test-1`) | `JUNK_SLUG_SUBSTRINGS` matches the slug → `evaluateJunkStation` returns `isJunk:true` → migration sets `noIndex:true` → station is dropped from sitemap and its SSR HTML emits `<meta name="robots" content="noindex, follow">`. |
| 3 | Codec / format / bitrate suffix records (`-mp3`, `-aac`, `-dab`, `-mp3-1`) | ~9 (e.g. `/lv/.../love-...-edam-dab`, `/de/sender/cleansingmusic-...-mp3`, `/hu/radio/war-fm-...-mp3-1`, `/id/stasiun/...-aac`) | `JUNK_SLUG_SUFFIXES` flags any slug ending in those tokens → noindex via the same path as row 2. |
| 4 | "-1" / "-2" collision-counter duplicates of a real station | several (`-mp3-1`, `-test-1`, etc.) | The migration script (`findDuplicateOfBaseSlug`) detects `-N` records whose base slug already exists as a sibling station and marks the dupe `noIndex:true`. We do **not** blanket-match `-(\d+)$` because many legitimate stations have numbers in their names (e.g. `Radio 100`, `FM 89-1`). |
| 5 | Song / program names sold as stations (`...phonk-remix`, `more-bounce-to-the-ounce`, `non-stop-music`, `only-mozart-mp3`) | 4 | `SONG_LIKE_NAME_PATTERNS` matches these against the lowercase station name → noindex. |
| 6 | Station translated into a language nobody in that region speaks (e.g. Mexican station rendered in Korean) | majority of remaining URLs (e.g. `/ko/스테이션/la-rancherita-los-mochis-...`, `/ur/اسٹیشن/exa-fm-...`, `/mr/steshan/exa-fm-...`, `/zh/电台/cjed-fm-...`) | New `getEligibleLanguages(station)` returns the country's primary language + diaspora languages + station broadcast languages + English. The sitemap (`server/routes/seo-sitemap-routes.ts`) only emits `<url>` and hreflang entries for eligible languages; the SSR renderer (`server/seo-renderer.ts`) sets `robots=noindex, follow` when the requested UI language is not eligible. |
| 7 | Bare `/sl` and `/da` language homepages with no localized content (TEMPORARY V2 threshold passed empty translations) | `/sl`, `/da` | `hasCompleteSeoTranslations()` in `shared/seo-config.ts` now strictly requires every key in **both** `REQUIRED_STATION_SEO_KEYS` (`default_station_about`, `from`, `genres`, `station_additional_info`, `live_radio`, `online_radio`, `radio_streaming`) **and** the new `REQUIRED_HOMEPAGE_SEO_KEYS` (`hero_worlds_best_radio`, `hero_over_100_countries`, `hero_listen_everywhere`, `nav_genres`, `nav_regions`, `nav_stations`, `popular_genres_title`, `popular_countries_title`). Languages missing any of those keys are dropped from the sitemap until translations land. |
| 8 | Weak meta description fallbacks | all stations without AI descriptions | `getStationMetaDescription` and `getLocalizedStationDescription` already produce a 2-sentence fallback (hook + amplify). The eligibility filter above ensures these fallbacks are only served in languages where the template translation keys exist. |

## Files changed

| File | Purpose |
|---|---|
| `server/seo/junk-station-rules.ts` (new) | `slugifyStationName`, `evaluateJunkStation` / `isJunkStation`, `getEligibleLanguages`, `isLanguageEligibleForStation` |
| `shared/mongo-schemas.ts` | `Station.slugAliases: string[]`, `Station.noIndex: boolean` |
| `shared/seo-config.ts` | `hasCompleteSeoTranslations` is now strict (was TEMPORARY V2 ≥1-key) |
| `server/routes/slug-routes.ts` | Both slug generators (`generateOptimizedUniqueSlug`, `generateUniqueSlug`) now call `slugifyStationName` |
| `server/routes/station-public-routes.ts` | Inline `generateUniqueSlug` now calls `slugifyStationName` |
| `server/routes/seo-sitemap-routes.ts` | Excludes `noIndex:true` stations, skips junk via `isJunkStation`, emits hreflang only for eligible languages |
| `server/seo-renderer.ts` | Falls back to `slugAliases` lookup (which now triggers a real 301 in `server/index-web.ts`); sets `robots=noindex, follow` when station is junk or current UI language is not eligible **and** points the canonical link at the eligible variant (English by default) so search engines consolidate ranking on the version we want indexed |
| `scripts/clean-content-quality-urls.ts` (new) | Backfill: re-slug every station, capture old slug in `slugAliases`, mark junk records `noIndex:true` (sibling-aware dupe detection), produce per-station CSV report |
| `scripts/audit-51-urls.ts` (new) | Reads `attached_assets/bing-content-quality-urls.txt` and writes `attached_assets/task-17-51url-status.csv` — one row per Bing URL with its post-change status (`indexable` / `noindex` / `301:<target>` / `removed:404` / `homepage-thin`) |
| `server/index-web.ts` | Issues `301` to canonical slug when SSR resolves a station via a `slugAlias` (real redirect, not just a content fallback) |

## How to apply in production

1. Deploy this task (schema fields are additive — `slugAliases` defaults `[]`,
   `noIndex` defaults `false`, both safe to roll forward).
2. Run the backfill once: `tsx scripts/clean-content-quality-urls.ts` (or
   `--dry-run` first to inspect `attached_assets/task-17-audit-report.csv`).
3. Trigger sitemap cache invalidation (existing admin endpoint) so the next
   crawl sees the cleaned URL set.
4. Re-submit the 51 URLs in Bing Webmaster (covered by the companion task #16).
